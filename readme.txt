## What we are building

A full-stack web application called **Drylining Estimator** — a tool for the construction industry that calculates materials needed to dryline (metal stud partitions + plasterboard) a room or building.

This is a **dissertation project** with a submission deadline of **27 April 2026**.

---

## Two core modes

### 1. Manual Estimator
User enters wall dimensions (length, height, openings like doors/windows) and gets a full material takeoff:
- Plasterboard sheets
- Steel studs (pieces + metres)
- Steel tracks (pieces + metres)
- Insulation packs
- Screws

### 2. AI Agent
User uploads a floor plan image or PDF. A Claude AI agent reads the drawing using vision, extracts wall dimensions, and calls the calculator to produce the estimate automatically.

---

## Key decisions made

- **Language:** Python (backend) + React/TypeScript (frontend)
- **Backend framework:** FastAPI
- **AI:** Claude API (Anthropic) — chosen for vision capability (reading floor plan images)
- **No custom LLM** — building your own LLM is not feasible; Claude is used as the engine, the agent logic and domain code is the original contribution
- **Calculator is a pure Python function** — the AI agent calls it as a tool, it does not recalculate anything itself
- **No Three.js in v1** — drop the 3D preview to save time, add later if time allows
- **Drop the fake confidence score** — the previous version had an arbitrary hardcoded score, remove it

---

## Why we are rebuilding from scratch

The previous version (`drylining_estimator/`) was mostly generated by ChatGPT. The ideas are original but the implementation needs to be owned by the student. Rebuilding from scratch means:
- Every line can be explained and defended in a dissertation viva
- Clean architecture from the start (no monolithic 500-line component)
- Proper separation of concerns
